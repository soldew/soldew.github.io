Determine source

 

If an extractor starts, the Determine source extractor step determines the source to be used for extraction.
This can be the OCR or the candidates of a previous extractor. The configuration of this extractor step specifies mainly

•Source to be used, i.e. OCR or candidates from a previous extractor

•Tokens where the labels and candidates are to be searched for, i.e. in single words, lines or paragraphs

•Properties for line and paragraph building
 

Settings for the extractor step
Ignore font size difference in paragraph building

Check box to specify whether the font size is to be considered for building paragraphs
 

Checked: Font size differences are ignored.
 

Unchecked: The font size is checked. If the font size difference of two words is greater than or equal to 2 pixels, it is assumed that they belong to different paragraphs.

 

Ignore font size difference in line building

 

Check box to specify whether the font size is to be considered for building lines

 
Checked: Font size differences are ignored.

 
Unchecked: The font size is checked. If the font size difference of two words is greater than or equal to 2 pixels, it is assumed that they belong to different lines.
 

Grouping for labels

Options to specify the parameters for line and paragraph building for the label search

Grouping level

The grouping level for labels determines where the extractor searches for the labels.

 

•Words: The extractor searches for single words that have more than one character. Words are separated by spaces.

•Lines: The extractor searches on lines.
It searches for labels in tokens that have the same Y coordinate and have no greater token distance than specified in the Token distance horizontal for line grouping and Token     distance vertical for line grouping parameters.

•Previous extractor: The extractor searches for the labels in the candidates of a previous extractor.

•Paragraphs: The extractor searches on paragraphs.
It searches on lines that have nearly the same X coordinate and that have no greater vertical distance than indicated in the Line distance vertical for paragraph grouping option.

•Sentences: The extractor searches on sentences.
It searches for labels in tokens that are between points considering that the first word of a sentence is capitalized. In addition, other layout-based sentence rules such as indention are taken into account.

•Lines from OCR Engine: The extractor searches on the lines that have already been determined by the OCR engine.
Since the OCR Engine's line building is done after paragraph building, this option allows to search on lines within columns if the page text is arranged in multiple columns. In case of line building by DocProStar (Lines option), a line is always built over the entire page width.

•Paragraphs from OCR Engine: The extractor searches on paragraphs that have already been determined by the OCR engine.

Vertical token distance for line grouping

Vertical line distance for paragraph grouping

Grouping for Candidates

 
Options to specify the parameters for line and paragraph building for the candidate search
 

Grouping level

 

The grouping level for candidates determines where the extractor searches for the candidates.

As with the grouping level for labels, candidates can be searched for in the candidates of a previous extractor or in tokens that contain just a word, a line or a whole paragraph. The options provided in the list box are the same as described above but apply to the search for candidates.
 

•Words: The extractor searches for single words that have more than one character. Words are separated by spaces.

•Lines: The extractor searches on lines.
It searches for labels in tokens that have the same Y coordinate and have no greater token distance than specified in the Token distance horizontal for line grouping and Token     distance vertical for line grouping parameters.

•Previous extractor: The extractor searches for the labels in the candidates of a previous extractor.

•Paragraphs: The extractor searches on paragraphs.
It searches on lines that have nearly the same X coordinate and that have no greater vertical distance than indicated in the Line distance vertical for paragraph grouping option.

•Sentences: The extractor searches on sentences.
It searches for labels in tokens that are between points considering that the first word of a sentence is capitalized. In addition, other layout-based sentence rules such as indention are taken into account.

•Lines from OCR Engine: The extractor searches on the lines that have already been determined by the OCR engine.
Since the OCR Engine's line building is done after paragraph building, this option allows to search on lines within columns if the page text is arranged in multiple columns. In case of line building by DocProStar  (Lines option), a line is always built over the entire page width.

•Paragraphs from OCR Engine: The extractor searches on paragraphs that have already been determined by the OCR engine.

Use of previous extractor candidates

 

Options to specify whether the candidates of the previous extractor will be added as new candidates to the extractor.
 

▪Add none as new candidates: No candidate is added.

▪Add all as new candidates: All candidates are added.

▪Add all except the best as new candidates: All but the best candidates are added.


Document pages

 
Specification of which Document pages the information is to be extracted from

The First and Last spin boxes make it possible to indicate the number of the first and the last Document pages. If, for example, in the First spin box the number 2 is chosen and in the Last spin box the number 2, the first 2 and the last 2 pages of the Document are used for extraction, all other Document pages are ignored.
 

Tip: If the exact number of the pages in the Document is not known, please ensure that the numbers in the spin boxes are high enough, e.g. 1000
